Overview
The AI Mentor is a chat‑based assistant that remembers each learner’s unique journey — strengths, weaknesses, study habits, and even emotional state. It uses forgetting‑curve theory to proactively suggest revision topics, generates personalised weekly reports, and keeps students motivated through streak tracking and momentum visualisations.

While the hackathon demo uses a rule‑based simulation (no external APIs), the architecture is explicitly designed to drop in any LLM with minimal code changes. This makes the Mentor instantly functional offline, yet infinitely scalable.

Features
Feature	Description
Contextual Memory	Remembers past conversations, struggled topics, and progress.
Forgetting‑Curve Alerts	Highlights topics that need revision based on last‑studied date.
Smart Revision Summaries	Generates concise summaries with key points, examples, and common mistakes.
Weekly Performance Report	Aggregates quiz scores, chat activity, and progress to give actionable advice.
Momentum Tracker	Visualises daily completion % against study windows to identify productive patterns.
Quick Replies	One‑click prompts for planning, motivation, placement tips, etc.
Chat History	Stores the last 15 messages for continuity across sessions.
Mark as Revised	Lets students confirm they’ve revised a topic, updating the AI’s memory.



Tech Stack
Frontend: Vanilla JavaScript, HTML5, CSS3 (no frameworks)

Visualisation: SVG (for momentum graphs)

State Management: Browser localStorage (for chat memory and history)

Mock AI Logic: Rule‑based if/else chain + static memory topics

Real AI Integration (optional): Any REST‑compliant LLM API (OpenAI, Gemini, Anthropic, etc.)

Deployment: Static files, runnable in any browser or web server




Architecture
Mock AI (Current – Hackathon Mode)
The current implementation uses no external API. All responses are generated by a decision tree inside the sendMessage() function:

javascript
// Simplified mock logic
if (lower.includes('sql')) {
  response = "🧠 I remember you struggled with SQL joins...";
} else if (lower.includes('plan')) {
  response = "Based on your progress, here's your plan...";
} else {
  // Rotate a set of memory phrases
  response = memories[memoryIndex % memories.length] + "...";
}
Static memory topics (memoryTopics object) store pre‑written content for revision summaries, but the system simulates “generation” by randomly shuffling excerpts.




✅ Why mock?

Zero latency – instant replies

No API keys or costs

Perfect for offline demos and hackathons

Demonstrates the UI/UX without backend dependencies

Real LLM Integration (Pluggable)
The architecture exposes a single entry point – the sendMessage() function. To replace the mock logic with a real AI, you only need to rewrite that function to call your preferred LLM API.

Example – OpenAI GPT‑4o:

javascript
async function sendMessage() {
  const msg = document.getElementById('chatInput').value.trim();
  if (!msg) return;
  addChatMessage('user', msg);

  const API_KEY = process.env.OPENAI_API_KEY; // use env variable
  const response = await fetch('https://api.openai.com/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${API_KEY}`
    },
    body: JSON.stringify({
      model: 'gpt-4o-mini',
      messages: [
        { role: 'system', content: 'You are a helpful AI Mentor for a student...' },
        { role: 'user', content: msg }
      ]
    })
  });
  const data = await response.json();
  const aiReply = data.choices[0].message.content;
  addChatMessage('ai', aiReply);
}
No other code changes – the rest of the UI (chat bubbles, memory badges, history) remains identical.

How It Works – Detailed Walkthrough
1. Chat Conversation
Each message is stored in the chatHistory array and displayed in the chat window.

The AI response is delayed by 600ms (for simulation) and tagged with a timestamp.

A “memory badge” (🧠 I remember you) appears on AI messages to reinforce the illusion of continuity.

2. Memory & Context
The mock AI maintains a rotating list of memory phrases (chatMemory array).

It also reads the student’s course progress (e.g., “Java 95%”, “DBMS 28%”) from the global state and weaves it into responses.

3. Forgetting‑Curve Alerts
The “Smart Memory Alert” panel lists topics with their last‑studied date and an urgency level (🔴 Urgent, 🟡 Consider).

Clicking “Quick Revision” generates a summary from the static memoryTopics object. The summary includes core concepts, examples, and common mistakes – all displayed in an expandable panel.

4. Weekly Report
The report analyses quiz scores from the LMS (simulated data) and chat history to recommend focus areas.

It is generated entirely client‑side, using hard‑coded sample scores for demonstration.

5. Momentum Graph
A pure‑SVG graph displays the last 7 days of completion % alongside the actual study time window.

The graph automatically computes a trend and assigns a status (✅ On track, ⚠️ Soft momentum, 📉 Dropping) with a custom message.

API Integration Guide
To replace the mock AI with a real LLM, follow these steps:

Obtain an API key from your provider.

Modify the sendMessage() function (around line 810 in the original file) to make a fetch request to the provider’s endpoint.

Pass the user’s message and any relevant context (e.g., current course progress) in the request body.

Parse the response and display it using addChatMessage('ai', response).

Handle errors gracefully (e.g., fallback to mock responses if the API fails).

Example with Google Gemini:

javascript
async function sendMessage() {
  const msg = document.getElementById('chatInput').value.trim();
  if (!msg) return;
  addChatMessage('user', msg);

  const API_KEY = 'YOUR_GEMINI_KEY';
  const response = await fetch(`https://generativelanguage.googleapis.com/v1beta/models/gemini-pro:generateContent?key=${API_KEY}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      contents: [{ parts: [{ text: msg }] }]
    })
  });
  const data = await response.json();
  const aiReply = data.candidates[0].content.parts[0].text;
  addChatMessage('ai', aiReply);
}
Note: In production, always store API keys on a server and proxy requests to avoid exposing them client‑side.
